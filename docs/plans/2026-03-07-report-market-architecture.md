## 1. პროექტის მიმოხილვა

**Report Market** — ამერიკის სატვირთო გადაზიდვების (Trucking) ინდუსტრიის მძღოლების რეპუტაციის პლატფორმა.

**პრობლემა**: ქერიერებსა და დისპეტჩერებს არ აქვთ ცენტრალიზებული სისტემა მძღოლების საიმედოობის შესამოწმებლად.

**გადაწყვეტა**: ნეგატიური რეპორტების სისტემა — მძღოლი იწყებს 100%-იანი რეიტინგით, ყოველი ვერიფიცირებული რეპორტი ამცირებს ქულას.

## 6. რეიტინგის სისტემა

### პრინციპი
- ყოველი მძღოლი იწყებს **100%**-იდან
- მხოლოდ **VERIFIED** რეპორტები მოქმედებს ქულაზე
- **DISPUTED** რეპორტი არ ითვლება გადაწყვეტამდე
- რეიტინგი გამოითვლება დინამიურად (არ ინახება DB-ში)

### სიმძიმის ქულები

**ტრანსპორტთან დაკავშირებული** (მაღალი გავლენა):

| შეცდომა | ქულის ჩამოჭრა | აღწერა |
|---------|-------------|--------|
| დაგვიანება (LATE_DELIVERY) | **-5** | ხშირი, ზომიერი გავლენა |
| ტვირთის დაზიანება (CARGO_DAMAGE) | **-10** | ფინანსური ზარალი |
| ავარია (ACCIDENT) | **-15** | უსაფრთხოება, ყველაზე სერიოზული |
| DOT ინსპექცია (DOT_INSPECTION_ISSUE) | **-8** | რეგულაციის დარღვევა |

**პროფესიონალური** (დაბალი, მაგრამ მნიშვნელოვანი):

| შეცდომა | ქულის ჩამოჭრა | აღწერა |
|---------|-------------|--------|
| კომუნიკაცია (COMMUNICATION) | **-3** | ხშირი, ნაკლებად სერიოზული |
| პუნქტუალურობა (PUNCTUALITY) | **-4** | ოპერაციული შეფერხება |
| დოკუმენტაცია (DOCUMENTATION) | **-5** | შესაბამისობის რისკი |
| უპასუხისმგებლობა (IRRESPONSIBILITY) | **-8** | ნდობის დარღვევა |

### ფერების კოდირება

```
  100% ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 80%
  🟢 Good Standing — კარგ მდგომარეობაშია

   79% ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 50%
   🟡 Needs Attention — ყურადღება საჭიროა

   49% ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  0%
   🔴 Critical — კრიტიკული მდგომარეობა
```

### გამოთვლის ფორმულა

```
რეიტინგი = max(0, 100 - Σ severity_weights)

მაგალითი:
  მძღოლს აქვს 3 VERIFIED რეპორტი:
  - დაგვიანება (-5)
  - კომუნიკაცია (-3)
  - ტვირთის დაზიანება (-10)

  რეიტინგი = 100 - (5 + 3 + 10) = 82% 🟢
```
